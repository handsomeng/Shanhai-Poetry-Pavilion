## 📋 核心改动

### 1. **移除 Tab 栏，单一主页面**
- ❌ 删除：2个Tab（学诗 + 诗集）
- ✅ 保留：诗集作为唯一主页面
- ✅ 好处：用户打开 App 直接看到自己的作品

### 2. **"学诗"功能调整为文字链接**
- ❌ 不再是独立 Tab
- ❌ 不是大卡片选项
- ✅ 改为底部小字链接："了解现代诗"（带下划线）
- ✅ 定位：轻量化学习入口，不抢主要操作的焦点

### 3. **创作模式选择器**
- **保持3个大卡片不变**：
  1. 🎯 主题写诗
  2. ✨ 临摹写诗
  3. 📝 自由写诗

- **底部新增文字链接**：
  - "了解现代诗"（小字 + 下划线）
  - 不占用卡片空间
  - 点击后全屏展示学习内容

### 4. **学习内容页面**
- 点击"了解现代诗"文字链接后
- 全屏展示原来的 6 个学习主题卡片
- 标题："了解现代诗"
- ❌ 移除副标题："从零开始，了解现代诗的创作技巧与美学"
- 带返回按钮

## 🎯 用户流程

### 场景1：我想写诗
```
打开App → 看到诗集 → 点击"写诗"按钮 → 选择模式 → 开始创作
```

### 场景2：我想学习
```
打开App → 看到诗集 → 点击"写诗"按钮 → 点击底部"了解现代诗"文字链接 → 查看学习主题
```

### 场景3：我想看我的作品
```
打开App → 直接看到诗集 ✅
```

## 📦 技术实现要点

1. **移除 MainTabView 的学诗 Tab**
   - 删除 `.learning` case
   - TabView 只保留 `.collection`（诗集）

2. **CreateModeSelectorView 底部添加文字链接**
   - 3个创作模式卡片保持不变
   - 底部新增 Button（文字样式）
   - 点击后跳转到学习页面

3. **LearningView 改造**
   - 标题改为"了解现代诗"
   - 移除副标题
   - 其他保持不变

4. **导航逻辑**
   - CreateModeSelectorView 添加 `@State` 控制学习页面显示
   - 使用 `.fullScreenCover` 或 `.sheet` 展示学习页面
