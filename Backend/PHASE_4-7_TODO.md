# Phase 4-7 剩余工作（简化版）

## Phase 4: 编辑已发布的诗 ⏳

**文件**：`ProfileView.swift` 中的诗集Tab

**要做的事**：
1. 点击已发布的诗 → 进入详情页
2. 详情页添加"编辑"按钮
3. 编辑保存后 → 自动重新提交审核
4. 审核通过 → 云端更新（点赞数保留）

**代码量**：约50行

---

## Phase 5: 诗集详情页按钮 ⏳

**文件**：需要创建 `PoemDetailEditView.swift`（或扩展现有的PoemDetailView）

**要做的事**：
1. 添加3个按钮：
   - **编辑** - 进入编辑模式
   - **发布到广场** - 如果未发布则发布
   - **删除** - 删除这首诗
2. 支持列表左滑删除

**代码量**：约100行

---

## Phase 6: 草稿页面优化 ⏳

**文件**：`ProfileView.swift` 中的草稿Tab

**要做的事**：
1. 点击草稿 → 直接进入编辑模式（不是只读）
2. 编辑后可以：
   - **保存到诗集** → 成功页面
   - **继续保存草稿** → 更新草稿

**代码量**：约30行

---

## Phase 7: 已发布Tab状态显示 ⏳

**文件**：`ProfileView.swift` 中的已发布Tab

**要做的事**：
1. 在诗歌卡片上显示审核状态角标：
   - 🟡 **审核中**
   - 🟢 **已通过**
   - 🔴 **被驳回**
2. 被驳回时点击显示驳回原因

**代码量**：约40行

## 我的建议

**优先做 Phase 5**（诗集详情页按钮）：
- 最常用
- 用户体验提升最大
- 包含编辑/删除核心功能

**其他可以暂缓**：
- Phase 4：复杂度高，先看用户是否真的需要编辑已发布的诗
- Phase 6-7：锦上添花，不影响主流程
